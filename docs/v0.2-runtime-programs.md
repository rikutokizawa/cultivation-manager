# v0.2 Runtime Programs

## 目的

Mac ローカルで完成させた backend を Raspberry Pi 5 へ移行する時に、必要な実行プログラムが既に揃っている状態にする。

## 追加したプログラム

### API

- `GET /dashboard/summary`
  - レコード数と最新更新時刻の要約を返す
- `POST /upload-image`
  - multipart で画像を受け取り、`image_records` へ登録する

### Services

- `backend/app/services/sensor_sources.py`
  - `dummy`
  - `json`
- `backend/app/services/camera_sources.py`
  - `dummy`
  - `directory`
- `backend/app/services/collection_pipeline.py`
  - センサ記録と画像記録の DB 保存
- `backend/app/services/image_storage.py`
  - アップロード画像の保存

### Scripts

- `backend/scripts/collect_sensor_data.py`
  - センサ収集を 1 回またはループ実行
- `backend/scripts/capture_images.py`
  - 画像取り込み / ダミー撮影を 1 回またはループ実行
- `backend/scripts/run_runtime.py`
  - センサ収集と画像取り込みを同時に定期実行

## Source 切替方針

### Sensor

- `dummy`
  - ローカル確認用のダミー値を生成
- `json`
  - `SENSOR_INPUT_JSON_PATH` の JSON 配列を取り込む

### Camera

- `dummy`
  - ダミー SVG 画像を生成
- `directory`
  - `INCOMING_IMAGE_PATH` から画像を拾って登録

## Raspberry Pi での使い方想定

### 最小構成

- backend API を起動
- `run_runtime.py` を常駐起動
- `SENSOR_SOURCE_TYPE=dummy`
- `CAMERA_SOURCE_TYPE=directory`

これで、センサはダミーのままでも、カメラ側だけ外部スクリプトや `libcamera-still` の保存先連携に差し替えやすい。

### 次段階

- `sensor_sources.py` に実センサ読み取りクラスを追加
- `camera_sources.py` に実カメラ撮影クラスを追加
- `systemd` サービスとして backend と runtime を常駐化
